# 字段映射 → 输出不同数据 的实现原理

> 本文专讲：财务在 Web 上**配置字段映射**后，系统是如何据此从 MySQL 事实表（或 legacy 导入行）里**算出不同指标值**的。

---

## 1. 核心思想

把「取数逻辑」存成**数据库配置**（`field_mappings` + `field_mapping_parts`）。计算引擎是**通用**的：按配置过滤、聚合、组合，不硬编码业务指标名。

```
配置（数据）          通用引擎（代码）              结果
field_mapping_parts  →  FieldAggregator  →  field_values {line_code: 数值}
```

**改口径 = 改配置，不改引擎代码。**

---

## 2. 三个概念

```
报表行 (field_mapping)           例：label「实际支付金额」line_code「mc_actual_payment」
  ├─ line_type=fetch             从数据取数（parts）
  ├─ line_type=formula           纯公式 expression
  └─ field_mapping_part ×N       每条 part 回答：从哪取、取哪些行、怎么聚合、怎么与上条组合
```

- **复用字段**：part 设 `ref_field_code`，规则在「基础取数字段」维护，当前行只引用。
- **组内多列**：同一 part 的 `sources[]` 多列先各自聚合再按 `combine_op` 组合。

---

## 3. 一条 part 的可配置项

| 配置项 | 字段 | 作用 |
|--------|------|------|
| 来源文件 | `source_file_keyword` | Catalog 文件关键字 |
| Sheet / 列 | `sheet_name` / `column_header` / `sources[]` | 定位数据列 |
| 聚合 | `aggregation` + `dedup_keys` | sum / count_distinct / sum_dedup … |
| 组合 | `combine_op` | add / subtract |
| 日期过滤 | `date_filter_column` + `date_format` | 只保留报表日 |
| 行筛选 | `row_filters` | eq/ne/in/contains/empty/between … |
| 排除样品 | `exclude_sample` | 整单 SKU 总额=0 |
| 排除刷单 | `exclude_review` | 不在刷单清单 |
| **排除当日退单** | `exclude_same_day_refund` | 当日下单且当日退款的 Order ID |
| 关联主表 | `join_to_orders` + `join_keys` | 佣金等跨表只保留有效订单行 |
| 仅样品 | `only_sample` | 样品单运费等 |
| 复用字段 | `ref_field_code` | 引用他行已配规则 |

## 4. Web → DB

### 前端（`mapping_modal.js`）

- 来源组：文件/Sheet/列、筛选弹窗、去重、高级规则勾选。
- 复用字段组：无高级规则（规则在基础取数字段）。
- 保存：`PUT /api/mappings/{id}`，body.parts 数组。

### 后端（`api.py`）

`_apply_parts`：删旧 parts 再按序重建 `FieldMappingPart`。

---

## 5. DB → 数值

入口：`report_engine.aggregate_field_values()`。

### 5.1 数据池

```python
# 优先 Catalog 事实表
rows, import_file_names = load_fact_rows(db, data_source_id, store_name)
context = build_daily_context(rows, data_source.config, report_date)
```

`DailyContext` 含：`sample_order_ids`、`review_order_ids`、`same_day_refund_order_ids`、`valid_order_keys` 等。

### 5.2 过滤（`_filter_rows`）

按 sheet、文件关键字、日期列、样品/刷单/当日退单排除、主表关联、`row_filters` 逐层筛选。

### 5.3 聚合与组合

`aggregate_part` → `combine_parts` → 得到 `field_values[line_code]`。

### 5.4 公式行

`mapping.expression` 中 `{field:code}` 代入后由 `formula.py`（AST 白名单）求值。

## 6. 端到端示例：「实际支付金额」

| 项 | 值 |
|----|-----|
| 文件 | 订单 |
| Sheet | OrderSKUList |
| 列 | Order Amount |
| 聚合 | sum_dedup，去重键 Order ID |
| 日期 | Created Time = 报表日 |
| 高级 | 排除样品、排除刷单 |

## 7. 触发「不同输出」的方式

| 目的 | 改什么 | 是否重导 ETL |
|------|--------|-------------|
| 换列头 | part.column_header | 否（Catalog 已有列） |
| 换口径 | aggregation / combine_op / row_filters | 否 |
| 排除当日退单 | exclude_same_day_refund | 否 |
| 换报表日 | 生成时选日期 | 否 |
| 源 Excel 内容变更 | — | **是**（重跑 ETL） |

## 9. 一句话总结

> **配置定义怎么取数，通用引擎照着算，公式把字段拼成报表。** 报表配置页与日报输出页共用同一套取数弹窗与 API。
