# 字段映射与报表模板合并方案

> **状态：已实施**（2026-06 起）  
> 报表行统一在 `field_mappings`；`/templates` 重定向至 `/mappings`；出报以数据源下 mappings 为准。

---

## 1. 决策摘要（已落地）

| 项 | 结论 |
|----|------|
| 公式行 | 与取数行同在「报表配置」；部分公式行已 `convert_formula_lines_to_fetch` 合并为取数+表达式 |
| 逻辑字段 | 降级为内部 `line_code`；`logical_field_id` 可空 |
| 数据源 | 每店铺一套 `field_mappings`（`data_source_id` 隔离） |
| 发布流程 | **取消** publish/draft，保存即生效 |
| 权限 | Demo：`accounts` / `stores` / Cookie 店铺隔离；`owner_id` 字段预留 |

---

## 2. 数据模型（当前）

```text
field_mappings（= 报表行）
├── data_source_id
├── line_type          fetch | formula | manual
├── label              指标名称
├── line_code          数据源内唯一，公式引用 {field:code}
├── report_group       分组
├── sort_order         顺序（日报页可拖动，导出按此排列）
├── expression         公式；fetch 行可为 {field:自身code}
├── format_type        usd | integer | percent
├── parts[]            field_mapping_parts（fetch 行）
└── owner_id           预留

仍保留只读/兼容：
├── report_templates / template_lines（种子、历史）
└── logical_fields（迁移期关联）
```

---

## 3. UI（当前）

| 页面 | 路径 | 能力 |
|------|------|------|
| 报表配置 | `/mappings` | 按店铺：日报字段表、基础取数字段、店铺设置、定时出报、配置导出 |
| 日报输出 | `/daily` | 生成日报、编辑值、拖动排序、导出 Excel / SKU 明细 |
| 取数弹窗 | 两页共用 `mapping_modal` | 来源组 + 复用字段 + 筛选 + 高级规则 |
| 刷单设置 | 两页共用 `review_settings_modal` | 清单导入 + 物流费按单固定 |

导航「报表模板」→ 302 至报表配置。

---

## 4. 引擎（当前）

```
generate_report_for_data_source(data_source_id, date, store)
  1. load_fact_rows + build_daily_context
  2. fetch 行 → aggregate → field_values[line_code]
  3. formula 行 → evaluate_expression
  4. report_values（line_label = mapping.label, sort_order = mapping.sort_order）
```

`build_grouped` / 导出 Excel 均按 `report_group` 与 `sort_order`，不再依赖 `TEMPLATE_GROUPS` 静态表。

## 6. 风险与缓解（经验）

| 风险 | 缓解 |
|------|------|
| line_code 变更导致公式失效 | 创建后不改 code，仅改 label |
| 迁移重复行 | `sync_report_lines(only_missing=True)` 幂等 |
| 两页弹窗分叉 | 共用 partial + 同一 API |

---

## 7. 不在本方案范围

- 越南店/多区域独立种子（需新 data_source）
- 平台费用明细列（规则文档附录，模板未建列）
