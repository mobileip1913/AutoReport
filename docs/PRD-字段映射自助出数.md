| 关联文档 | `docs/PRD.md`、`docs/字段映射与取数实现.md` |

---

## 1. 解决什么问题

财务要把多份平台 Excel 汇总成日报。传统方式口径绑死在 Excel/脚本里，列头一变就要找研发。

**本功能目标**：在网页上配置「指标从哪取、怎么算」，保存即生效，下次出报自动按新口径计算。

---

## 2. 核心概念

```
指标（报表行 label）     例：实际支付金额
  └─ 取数规则 parts      文件·Sheet·列·聚合·筛选·高级规则
       └─ 可多条 ±       例：应收 = SKU折后 + SKU平台折扣
  └─ 或复用他字段        ref_field_code（规则在基础取数字段）
```

---

## 3. 关键场景

| 场景 | 操作 |
|------|------|
| 列头变更 | 配置弹窗改列头 → 保存 → 重新出报 |
| 多列组合 | 同一指标添加多条 part，± 组合 |
| 行筛选 | 来源组「筛选」弹窗：含/不含/为空等 |
| 排除样品/刷单/当日退单 | 高级规则勾选（复用字段卡片无此项） |
| 跨表佣金 | 勾选「关联数据主表有效行」+ 匹配键 |
| 刷单指标 | 「刷单设置」导入清单；物流费店铺级按单固定 |
| 调整列顺序 | 日报输出页拖动 → 导出 Excel 列位同步 |

---

## 4. 功能详述

### 4.1 报表配置页（`/mappings`）

- 按店铺展示日报字段表：规则摘要、配置/刷单设置入口。
- **基础取数字段**：供组合指标 `ref_field_code` 引用。
- **店铺设置**：日期主表（订单文件/Sheet/日期列）、每日自动生成时间、Excel 模板绑定。
- **配置导出 JSON**：含 mappings、settings、catalog 快照。

### 4.2 配置弹窗（`mapping_modal.js`）

| 配置项 | 说明 |
|--------|------|
| 来源文件 / Sheet / 列 | 可搜索下拉，来自 Catalog + schema API |
| 组内多列 | `sources[]`，先聚合再组合 |
| 筛选 | 每来源组独立 `row_filters` |
| 去重 | 去重键 + 聚合方式联动 |
| 高级规则 | 排除样品、排除刷单、**排除当日退单**、关联主表 |
| 复用字段 | 仅选字段 code，不编辑 part |

保存：`PUT /api/mappings/{id}`；校验 Sheet/列完整。

### 4.3 日报输出（`/daily`）

- 选店铺、日期 → 生成日报。
- 编辑报表值（手工行/覆盖）、拖动字段排序。
- 导出 Excel（按 `sort_order`）、导出 SKU 明细。
- **与报表配置共用**取数弹窗、刷单设置、规则摘要展示。

### 4.4 自动出数

- 数据来自 MySQL `fact_*`（ETL 维护）。
- `build_daily_context` 计算样品/刷单/当日退单集合。
- 试点店 2026-06-22 核心指标已与独立脚本对账一致。

## 5. 业务约定

| 规则 | 说明 |
|------|------|
| 配置是规则不是结果 | 改配置影响下次出报，历史 run 不变 |
| Excel 内容变更 | 需重跑 ETL，非 Web 重导 |
| 取消类指标 | 实现为 Created Time + Order Status=Canceled（与部分规则文档 Cancelled Time 不同） |
| 占位指标 | 物流费/产品成本/固定费用/框返等无源时为 0 |
| 利润/总利润 | 手工填写，导出留空 |
