## 1. 这个功能解决什么问题

### 1.1 现状痛点

财务每天要把 RPA 下载的多个 Excel 汇总成日报。今天的麻烦在于：

- **口径写死在 Excel 公式/脚本里**：平台把列头从 `Order Amount` 改成别的，公式全废。
- **改口径要排期**：财务想加一个指标、改一个算法，得提需求 → 研发改代码 → 上线，几天起步。
- **跨人跨店口径不一**：每个人的 Excel 都不一样，没法对比、难追溯。

### 1.2 本功能的目标

> 让财务在网页上**自己配置「某个指标从哪取、怎么算」**，配完即生效，下次出报自动按新口径计算——**全程不找研发**。

一句话：**改映射，自动出数据。**

## 3. 核心概念（给财务的认知模型）

财务只需理解三层，像「搭积木」：

```
指标（我要的结果）         例：实际支付金额、应收金额、达人佣金
  └─ 取数规则（怎么来的）   从「哪个文件·哪个表·哪一列」取，要不要去重、过滤、排除
       └─ 可多条相加减      例：应收 = SKU折后小计 ＋ SKU平台折扣
```

- **指标**：报表上要看的一行数（系统里叫「逻辑字段」）。
- **取数规则**：告诉系统这行数从哪来、怎么算。一个指标可以由**多条规则加减**得到。

财务**不需要懂数据库、不需要写代码**，全部在弹窗里点选完成。

## 4. 关键场景

### 场景 A：平台改了列头（最高频）

> TikTok 把订单金额列从 `Order Amount` 改名为 `Order Amount1`。
> 财务进入「字段映射」→ 找到「实际支付金额」→ 把列头改成 `Order Amount1` → 保存。
> **报表模板的公式一行不用动**，重新出报即按新列计算。

### 场景 B：新增/调整一个指标算法

> 主管要求「应支付金额」口径调整为：实际支付 ＋ 支付平台折扣 ＋ SKU平台折扣。
> 财务在该指标下配 3 条规则，组合运算选「＋」，保存生效。

### 场景 C：加一个筛选条件

> 只统计「已完成」状态、且非样品单的订单。
> 财务在规则的「高级规则」里加行过滤条件 `Order Status = 已完成`，勾选「排除样品单」。

### 场景 D：跨文件组合

> 净费用 = 订单表某列 − 结算表的扣费列。
> 配两条规则：规则1 来源文件「订单」加上，规则2 来源文件「结算表」减去。

## 5. 功能需求详述

### 5.1 字段映射列表页

**展示**：每个数据源下，各指标当前的取数规则摘要（文件/表/列/聚合方式一目了然）。

**操作**：
- 「+ 新增映射」：为某数据源的某指标新建取数规则。
- 每行「配置」：编辑已有映射。

**验收**：
- [ ] 列表按数据源 + 指标展示，规则摘要清晰可读（含多条 part 的加减关系）。
- [ ] 未配置的指标有明确「未配置」提示。

### 5.2 配置弹窗（核心交互）

每条取数规则可配置：

| 配置项 | 说明 | 必填 |
|--------|------|------|
| 来源文件关键字 | 多文件时选哪个文件（订单/结算/退货…），空=全部 | 否 |
| Sheet | 取哪个工作表 | 是 |
| 列头 | 取哪一列 | 是 |
| 列头别名 | 防列头变动，多写几个名字任一命中即可 | 否 |
| 聚合方式 | 求和 / 计数 / 去重计数 / 去重求和 / 取最大 / 平均 | 是 |
| 去重键 | 去重依据（如按订单号去重） | 否 |
| 组合运算 | 与上一条规则：＋加上 / −减去 | 是 |
| **日报高级规则** | 折叠区，见下 | 否 |

**日报高级规则（折叠）**：
- 按日期列过滤（取值 = 报表日期）+ 日期格式（自动/美式/欧式/ISO）
- 排除样品单、排除刷单单、关联当日有效订单（勾选）
- 行过滤条件（列 + 运算符 + 值，可多条）

**交互要求**：
- [ ] Sheet、列头、日期列均为**可搜索下拉**，从真实文件结构读取，不靠手记。
- [ ] **打开弹窗实时刷新列结构**：财务在 Excel 新增列后，打开配置即可看到新列（无需先重新导入也能选中配置）。
- [ ] 支持「+ 添加规则」增加多条 part，可删除。
- [ ] 保存校验：每条规则必须有 Sheet 和列头，否则提示。
- [ ] 保存成功后弹窗**淡出关闭**并刷新列表（有过渡动画）。

**验收**：
- [ ] 配置保存后立即生效于下一次出报，不影响历史报表。
- [ ] 删除映射、取消、点遮罩均能正常关闭弹窗。

### 5.3 自动出数据（报表生成）

**流程**：财务在「日报输出」选数据源 + 报表日期 → 生成 → 系统按已配置的映射自动算出全部指标 → 按日报模板分组展示 → 可导出 Excel。

**计算行为（财务可感知的口径）**：
- 按规则里的「日期列」筛选当日数据。
- 「去重求和/去重计数」按去重键合并（同一订单多行只算一次）。
- 勾选了「排除样品/刷单」的指标会自动剔除对应订单。
- 多条规则按「＋/−」组合成最终值。

**验收**：
- [ ] 生成结果与财务手算口径一致（试点店已对账：实际支付、应收、订单数完全一致）。
- [ ] 缺数据/除零等异常有友好提示，不崩溃。
- [ ] 导出的 Excel 结构与「日报模板」一致。

### 5.4 映射告警

导入数据时，若某指标期望的列在文件里找不到，记录告警（不阻断导入），财务可在「映射日志」定位到具体指标、Sheet、实际列头清单。

**验收**：
- [ ] 列头不匹配产生 warning 日志，含字段名、期望列、实际列头列表。

## 6. 关键业务规则（财务必须知道的约定）

| 规则 | 说明 |
|------|------|
| **配置存的是「规则」不是「结果」** | 改完对下次出报生效，已生成的历史报表不变。 |
| **改了 Excel 内容必须重新导入** | 系统算的是导入时的数据快照；改了文件不重导，数据不会变。 |
| **能在下拉选到新列 ≠ 有数据** | 新列要参与计算，仍需重新导入该文件。 |
| **占位指标暂为 0** | 刷单/样品/广告/成本等数据源未接入前，相关指标显示 0，接入后自动生效。 |

## 7. 典型操作路径（端到端）

```
1. 数据导入：上传/导入 RPA 的 Excel
        ↓
2. 字段映射：为每个指标配置取数规则（本功能核心）
        ↓
3. 日报输出：选数据源 + 日期 → 生成 → 查看分组报表
        ↓
4. 导出 Excel：按日报模板格式下载
   （口径要变？回到第 2 步改映射，第 3 步重新生成即可）
```

## 9. 范围边界

### 本期做（已验证）
- 字段映射在线增删改、多条规则加减组合。
- 高级规则：日期过滤、行条件、样品/刷单排除、跨表关联。
- 列结构实时刷新、自动出报、导出 Excel。

### 暂不做（后续演进）
- 刷单/广告/成本等数据源的导入（占位中）。
- 定时自动出报（当前手动触发）。
- 公式编辑器可视化、模板版本回滚、权限隔离。
