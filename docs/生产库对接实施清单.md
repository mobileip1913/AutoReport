# 生产库对接实施清单

> 目标：将生产库 `eb_overseas_tk_*` 表结构与数据导入本地 MySQL，AutoReport 读数/ETL/日报全链路适配。  
> 对照文档：[数据库对比说明.md](../sql/数据库对比说明.md) · DDL：[跨境订单建表语句_v2.sql](../sql/跨境订单建表语句_v2.sql)

---

## 一、前置条件

- [ ] `.env` 中 `DATABASE_URL` 指向 **MySQL 5.7+**（非 SQLite）
- [ ] 已获得生产库 DDL 与（可选）数据 dump
- [ ] 确认本店铺在生产侧的 `store_id`、`shop_code`（写入 `stores` 或 `data_sources.config`）

---

## 二、数据库（表结构完全一致）

| 步骤 | 命令 / 操作 | 状态 |
|------|-------------|------|
| 2.1 | 执行生产 DDL：`python scripts/apply_production_ddl.py` | ☐ |
| 2.2 | 若表已存在且需保留数据：`python scripts/apply_production_ddl.py --no-drop` | ☐ |
| 2.3 | 校验 6 张表存在：`eb_overseas_tk_order` / `_order_return` / `_affiliate_creator` / `_affiliate_partner` / `_finance_statements` / `_finance_on_hold` | ☐ |
| 2.4 | **方式 A**：从生产 mysqldump 导入（推荐，数据与线上一致） | ☐ |
| 2.5 | **方式 B**：本地 Excel ETL：`python scripts/import_meichong.py`（写入生产列格式） | ☐ |

### 生产数据导入示例（方式 A）

```bash
# 生产侧导出（示例）
mysqldump -h <host> -u <user> -p <db> \
  eb_overseas_tk_order eb_overseas_tk_order_return \
  eb_overseas_tk_affiliate_creator eb_overseas_tk_affiliate_partner \
  eb_overseas_tk_finance_statements eb_overseas_tk_finance_on_hold \
  > production_tk_facts.sql

# 本地（先 DDL 再导入数据）
python scripts/apply_production_ddl.py --no-drop
mysql -h <host> -u <user> -p <local_db> < production_tk_facts.sql
```

## 四、代码适配项（本仓库已实现）

| 模块 | 变更要点 |
|------|----------|
| `app/services/production_schema.py` | 从 DDL 解析 COMMENT→db_column |
| `app/services/production_fact.py` | store_id 解析、extra_data 展开、Excel→生产行 |
| `app/services/fact_provider.py` | 生产表按 `store_id` 查询 |
| `app/services/excel_reader.py` | 重复列头追加「 2」与生产 `_2` 列对齐 |
| `app/services/fact_storage.py` | `clear_production_fact_rows` |
| `app/models.py` | `Store.production_store_id`、`shop_code`；`EtlBatch.excel_order_id` |
| `scripts/etl/meichong_dataset.py` | 6 张生产表配置 |
| `scripts/import_meichong.py` | Excel → `eb_overseas_tk_*` |
| `app/services/catalog_cleanup.py` | 启动时清理 legacy Catalog |

上生产：**仅修改 `.env` 的 `DATABASE_URL`** 指向生产 MySQL（库中需已有 `eb_overseas_store` + 六张事实表及数据）。
