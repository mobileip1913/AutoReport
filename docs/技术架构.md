# AutoReport 技术架构说明

> 面向研发的实现同步文档。配套产品文档见 `docs/PRD.md`。
> 本文描述当前主开发路径（`app/`，Python + FastAPI）的真实实现，已跑通美宠 TikTok 美国本土店日报；生产数据走 **MySQL 事实表**。

---

## 1. 一句话概述

把 RPA/ETL 同步到 MySQL 的店铺业务数据，财务在 Web 上**按店铺自助配置「报表行 → 取数规则 / 公式」**，系统按日期聚合算出日经营报表并导出 Excel。核心价值：**列头/口径变更不再依赖研发改代码**。

---

## 2. 技术栈

| 层 | 技术 | 说明 |
|----|------|------|
| Web 框架 | FastAPI | HTML 页面路由 + JSON API |
| ASGI | Uvicorn | 默认 `0.0.0.0:8081` |
| ORM | SQLAlchemy 2.0 | 声明式模型 + Session |
| 数据库 | **MySQL**（生产） | `.env` → `DATABASE_URL`；本地可临时 SQLite |
| Excel | openpyxl | 解析源文件（ETL）、导出日报 |
| 模板 | Jinja2 | 服务端渲染 |
| 前端 | Tailwind CDN + 原生 JS | `app/static/*.js`，无构建步骤 |
| 定时任务 | APScheduler | 按店铺 `daily_generate_at` 自动生成昨日日报 |
| 配置 | pydantic-settings | `.env` → `Settings` |

**刻意保持轻量**：无 Vue/React、无前端构建、无 Celery。

## 4. 数据模型（核心表）

### 4.1 业务数据（出报取数）

```
catalog_files / catalog_sheets / catalog_columns   Excel 概念 → 事实表与列映射
fact_*（如 fact_mc_order_sku）                      按店铺存储的业务行
```

`FactProvider` 按 Catalog 对每张事实表执行固定模式 `SELECT ... WHERE data_source_id=? AND store_name=?`，再在内存中按 `field_mapping_parts` 过滤聚合。

### 4.2 配置与报表

```
data_sources              数据源；config(JSON) 存日期主表、刷单清单、定时出报时间等
  └─ field_mappings       报表行（fetch 取数 / formula 公式），含 label、line_code、sort_order、report_group
       └─ field_mapping_parts   单条取数规则（可多条加减、可复用字段 ref_field_code）
stores / accounts         Demo 店铺与账号；Cookie 隔离可访问店铺
report_runs / report_values   一次出报与各指标值（可手工覆盖）
```

`report_templates` / `template_lines` **仍保留**（种子/兼容），日常出报以 **`field_mappings` 为准**；`/templates` 已重定向到报表配置。

### 4.3 兼容层（Legacy）

```
data_imports / data_rows    旧版 Web 上传 Excel 入库；无 Catalog 时回退此路径
logical_fields              逻辑字段注册表；新行以 mapping.label + line_code 为主
mapping_logs                列头不匹配等告警
```

### 4.4 `field_mapping_parts` 关键字段

| 字段 | 作用 |
|------|------|
| `source_file_keyword` / `sheet_name` / `column_header` / `sources[]` | 定位文件、Sheet、列（组内可多列） |
| `aggregation` / `dedup_keys` | sum / count_distinct / sum_dedup 等 |
| `combine_op` | 与上一条 part：add / subtract |
| `date_filter_column` + `date_format` | 行级日期 = 报表日 |
| `row_filters` | 行筛选（eq/in/contains/empty 等） |
| `exclude_sample` / `exclude_review` / `exclude_same_day_refund` | 排除样品 / 刷单 / 当日退单 |
| `join_to_orders` + `join_keys` | 关联日期主表有效行 |
| `ref_field_code` | 复用同店已配字段（规则在基础取数字段维护） |
| `only_sample` | 仅样品单（如样品运费） |

## 5. 核心流程

### 5.1 数据入湖（ETL，非 Web 上传）

```
files/*.xlsx → scripts/import_meichong.py → catalog_* + fact_*（MySQL）
```

Web 端**不再上传 Excel**；运维/RPA 维护 MySQL。开发环境可用脚本灌库。

### 5.2 报表生成

```
选店铺(data_source) + 报表日期
   │
   ├─ load_fact_rows → 事实表行（或 legacy data_rows）
   ├─ build_daily_context → 样品集、刷单集、当日退单 Order ID、有效订单键
   │
   ├─ 对每个 fetch 行：parts 过滤 → 聚合 → 写入 field_values[line_code]
   ├─ 对 formula 行：evaluate_expression（{field:code}）
   │
   └─ 写 report_runs / report_values（sort_order 来自 field_mappings）
```

### 5.3 Excel 导出

- 复制 `files/日报模板.xlsx`（或店铺绑定模板）
- **按 `field_mappings.sort_order` 从 F 列起依次写入**表头与数据（与日报页拖动排序一致）
- 合并单元格表头写入第 2 行锚点；清除尾部 AF+ 旧版残留列

## 6. 页面与权限（Demo）

| 路径 | 说明 |
|------|------|
| `/` | 概览 |
| `/mappings` | **报表配置**：取数规则、公式、分组、店铺设置、定时出报、配置导出 JSON |
| `/daily` | **日报输出**：字段排序、手工改值、生成/导出 Excel、导出 SKU 明细 |
| `/templates` ` /reports` ` /logs` ` /imports` | 重定向到 mappings 或 daily |

## 7. 主要 API

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | `/api/data-sources/{id}/schema` | Sheet/列结构（Catalog + 源文件） |
| GET/PUT | `/api/mappings/{id}` | 取数行 CRUD |
| PUT | `/api/data-sources/{id}/report-fields/order` | 日报字段排序 |
| GET/PUT | `/api/data-sources/{id}/settings` | 日期主表、定时出报、刷单物流费等 |
| POST | `/api/data-sources/{id}/review-orders/import` | 导入刷单清单 |
| GET | `/api/data-sources/{id}/config/export` | 导出店铺配置 JSON |
| PATCH | `/api/report-runs/{id}/values/{value_id}` | 日报值手工覆盖 |
| POST | `/daily/generate` | 生成日报 |
| GET | `/daily/{run_id}/export` | 导出 Excel |
| GET | `/daily/{run_id}/export-sku` | 导出 SKU 销量明细 |

## 9. 运行与初始化

```bash
python -m venv .venv
.venv\Scripts\activate
pip install -r requirements.txt
copy .env.example .env   # 配置 MySQL DATABASE_URL
uvicorn app.main:app --reload --host 0.0.0.0 --port 8081
```

启动时（`main.py`）：`run_migrations` → 注册美宠数据源 → 种子规则 → `sync_report_lines` → 启动 APScheduler。

可选灌库：`python scripts/import_meichong.py --force`；对账：`python scripts/verify_meichong.py 2026-06-22`。

## 10. 现状与演进

### 已验证

- 美宠 6 类 Excel ETL 入 MySQL；Web 配置取数、刷单导入、定时出报、日报编辑与导出。
- 报表配置与日报输出共用取数弹窗；字段拖动排序与导出列位一致。

### 已知口径约定（研发需注意）

1. **取消类指标**：实现为 `Created Time=日报` + `Order Status∈{Canceled,Cancelled}`（与规则文档 `Cancelled Time` 不同，见业务确认）。
2. **退款金额**：退货表 `Return unit price` 求和（近似，非严格 SKU 总额）。
3. **店铺佣金**：结算 `Fees` 全量求和（未扣达人佣金/运费）。
4. **刷单物流费**：店铺设置按单固定 × 不重复 Order ID（可排除当日退单）；与 part 级 `exclude_same_day_refund` 独立。
5. **占位字段**：物流费/产品成本/固定费用/框返等无数据源时出报为 0。

### 待演进

| 主题 | 建议 |
|------|------|
| 正式登录 | 替换 Demo Cookie 为 SSO/JWT |
| 越南/多区域店 | 新建 data_source + Catalog，核对结算列名 |
| 广告费 | 规则文档要求导入；当前部分从结算表 GMV Max 字段汇总 |
| 大数据量 | 按日预聚合 / 分析库 |
| 公式 | 扩展 HyperFormula（VLOOKUP 等） |
