# AutoReport 技术架构说明（Demo v0.1）

> 面向研发的实现同步文档。配套产品文档见 `docs/PRD.md`。
> 本文描述当前 Demo 的真实实现（已跑通美宠 TikTok 美国本土店日报）。

---

## 1. 一句话概述

把 RPA 导出的多个 Excel 解析入库，财务在 Web 上**自助配置「逻辑字段 → 取数规则」**，系统按报表模板的公式自动算出日经营报表并可导出 Excel。核心价值：**列头/口径变更不再依赖研发改代码**。

## 4. 数据模型（11 张表）

```
data_sources         数据源（平台/店铺）  ── config(JSON) 存日报上下文
  └─ data_imports    每次导入记录（文件名/行数/日期/店铺）
       └─ data_rows  解析后每一行：sheet_name + row_data(JSON)
  └─ field_mappings  数据源级「逻辑字段 → 取数」映射
       └─ field_mapping_parts  单条取数规则（可多条加减组合）
mapping_logs         列头不匹配等告警
logical_fields       跨平台统一指标（如 mc_actual_payment）
report_templates     报表模板
  └─ template_lines  指标行（label + 公式表达式）
report_runs          一次报表生成
  └─ report_values   各指标计算结果
```

### 关键设计：`data_rows.row_data` 用 JSON 存整行

```python
# 一行 Excel = 一条记录，列头→值打包成 JSON
row_data = {"Order ID": "577...", "Order Amount": 27.23, "Created Time": "06/22/2026 2:02:32 PM", ...}
```

**为什么**：不同平台/文件列头千差万别（订单 60 列、结算 100+ 列），列结构不固定。用 JSON 存可做到**加列、改列名都不用改数据库表结构**——这是「财务自助、研发少介入」的技术基础。

### 关键设计：`field_mapping_parts` 是规则的最小单元

一个逻辑字段（如「应收金额」）可由**多条 part 加减组合**，每条 part 支持：

| 字段 | 作用 |
|------|------|
| `source_file_keyword` | 按文件名关键字选文件（订单/结算/退货…） |
| `sheet_name` / `column_header` / `aliases` | 定位 Sheet + 列（别名防列头漂移） |
| `aggregation` | sum / count / count_distinct / sum_dedup / max_dedup / avg |
| `dedup_keys` | 去重键（如按 Order ID 去重） |
| `combine_op` | 与上一条 part 的运算：add / subtract |
| `date_filter_column` + `date_format` | **行级日期过滤**（取值=报表日期），支持 us/eu/iso 混用 |
| `row_filters` | 行条件过滤（eq/in/gt/nonempty 等） |
| `exclude_sample` / `exclude_review` | 排除样品单 / 刷单单 |
| `join_to_orders` | 跨表关联：只保留命中当日有效订单的行 |

## 5. 核心流程

### 5.1 数据导入

```
Excel → openpyxl 读取 → 智能识别表头行 → 跳过说明行 → 逐行打包 JSON → 批量写 data_rows
                                                          └→ 列头不匹配写 mapping_logs（告警，不阻断）
```

### 5.2 报表生成（日报模式）

```
选数据源 + 日期
   │
   ├─ 跨所有导入按店铺取出全部 data_rows（月度数据池）
   ├─ build_daily_context：算出样品订单集、刷单订单集、当日有效订单键
   │
   ├─ 对每个 field_mapping：
   │     每条 part 过滤行（文件/Sheet/日期/排除/关联）→ 聚合 → part 间加减组合
   │     ⇒ 得到 field_values = {逻辑字段code: 数值}
   │
   └─ 对模板每个 template_line：
         用 formula 引擎把 {field:xxx} 代入表达式 → 求值 → 格式化
         ⇒ 写 report_values
```

## 6. 关键技术点（研发重点关注）

### 6.1 智能表头识别（`excel_parser.py`）

真实 RPA 文件的第一行常常**不是列头**：未结算单首行是免责声明、联盟服务商首行是 Note、订单表第 2 行是字段说明。

- **表头定位**：前 15 行里「非空单元格最多」的行即表头。
- **说明行识别**：用下方数据行确定哪些列是数值列；若紧跟表头的那行在这些列上多为文本，则判定为说明行并跳过。

### 6.2 日报规则引擎（`field_aggregator.py`）

把《美宠日报规则-TK》的口径转成可配置的行级计算：

- **多格式日期解析**：订单 `MM/DD/YYYY`、退货/联盟 `DD/MM/YYYY`、结算 `YYYY/MM/DD` 混用，统一归一化。
- **金额清洗**：`"$19.04"`、千分位逗号、括号负数等都能转成数值。
- **样品单识别**：同一订单 `SKU Subtotal After Discount + SKU Platform Discount = 0`。
- **去重聚合**：`sum_dedup`（同订单只算一次）、`count_distinct`（订单数）。
- **跨表关联**：达人佣金按 `Order ID & SKU ID` 关联订单表中「当日有效订单」。

### 6.3 安全公式引擎（`formula.py`）

- 语法：`{field:code}` 引用字段；`=` 开头为算术表达式（+ - * /）。
- 实现：正则替换字段值 → Python `ast` 解析 → **白名单 AST 节点求值**（不用 `eval`，杜绝注入），除零等异常有友好报错。

### 6.4 列结构实时刷新（`schema.py`）

下拉列表的列名来自「已导入数据」+「**源文件当前表头实时合并**」。所以财务在 Excel 新增列后，打开配置弹窗即可看到新列（`/api/data-sources/{id}/schema` 实时读文件）。
⚠️ 注意：能选到新列 ≠ 有数据，新列要参与计算仍需**重新导入**。

### 6.5 时区（`timezone.py`）

数据库统一存 UTC，展示时经 Jinja `cst` 过滤器转东八区。新旧数据都正确，无需迁移。

## 8. 主要接口

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | `/` `/templates` `/mappings` `/imports` `/daily` `/reports` `/logs` | 页面 |
| POST | `/daily/generate` | 生成日报 |
| GET | `/daily/{run_id}/export` | 导出日报 Excel |
| GET | `/api/data-sources/{id}/schema` | 实时列结构（Sheet/列/文件） |
| GET/POST/PUT/DELETE | `/api/mappings[/{id}]` | 字段映射增删改查 |
| POST | `/api/import` | 上传 Excel 导入 |
| POST | `/api/generate` | 按模板生成报表 |

## 9. 运行与初始化

```bash
python -m venv .venv
.venv\Scripts\activate
pip install -r requirements.txt
uvicorn app.main:app --reload --port 8000   # http://localhost:8000
```

启动时（`main.py` 的 startup）自动：迁移建表 → 注册美宠数据源 → 补逻辑字段 → 写入美宠日报规则与模板。
导入真实数据：`python scripts/import_meichong.py`；口径对账：`python scripts/verify_meichong.py 2026-06-22`。

## 10. 现状与生产演进建议

### Demo 已验证
- 6 个真实 Excel 共 5.3 万行入库；按规则算出日报，**独立重算对账完全一致**。
- 字段映射可视化增删改、日报输出与 Excel 导出。

### 待补 / 生产演进（供排期参考）

| 主题 | Demo 现状 | 生产建议 |
|------|-----------|----------|
| 数据库 | SQLite | MySQL（已装驱动，改 `DATABASE_URL` 即可） |
| 数据导入 | 手动/脚本 | RPA 对接：目录监听 / API 自动导入 |
| 刷单/广告/成本 | 占位字段（=0） | 新增导入表，填充 `review_order_ids` 等 |
| 定时出报 | 手动触发 | APScheduler / Celery 日批 |
| 权限 | 无 | 登录 + 店铺级数据隔离 |
| 公式能力 | 四则运算 | 引入 HyperFormula（VLOOKUP 等） |
| 大数据量 | 全量加载聚合 | 按日预聚合 / ClickHouse 历史层 |
| 前端 | SSR + 原生 JS | Vue/React + 组件库（视需要） |

### 已知约定（研发需注意）
1. **改了 Excel 必须重新导入**，库里 JSON 是导入时快照。
2. 同一「数据源+文件名」重导会先删旧导入再写（见 `scripts/import_meichong.py --force`）。
3. 占位指标（刷单/样品/广告/成本/固定费用/框返）当前恒为 0，接入数据后自动生效。
4. 利润为**透明估算口径**，公式写在模板指标行里，可在模板页调整。
