| 更新日期 | 2026-07-07 |

## 1. 背景与问题

### 1.1 业务背景

公司拥有多家跨境电商店铺（当前试点：**美宠 · TikTok 美国本土店**）。RPA 每日导出多类 Excel（订单、退货、结算、联盟佣金等），需汇总为统一口径日报。

### 1.2 核心问题

| 问题 | 影响 |
|------|------|
| 口径绑死在 Excel/脚本 | 列头一变公式全废 |
| 改需求依赖研发 | 周期长 |
| 跨店口径不一致 | 难对比、难审计 |

### 1.3 产品目标

1. 财务在 Web **按店铺**配置报表行（取数 + 公式 + 顺序 + 分组）
2. 系统从 **MySQL 事实表**按日聚合出报
3. 列头/规则变更以配置为主，研发只维护 ETL 与平台适配
4. 支持日报手工调整、导出 Excel、定时自动生成

## 2. 用户与场景

| 角色 | 诉求 |
|------|------|
| 财务专员 | 自助改口径、出日报、导出 |
| 财务主管 | 口径一致、可审计 |
| 研发/运维 | 维护 ETL、Catalog、种子规则 |
| 管理员 | 店铺/账号（Demo 已有多店隔离雏形） |

**典型场景**

- **列头变更**：改映射列头 → 保存 → 重新生成日报。
- **新增指标**：日报页「添加字段」→ 配置取数 → 拖动排序 → 导出。
- **刷单**：导入刷单清单；物流费在「刷单设置」按单固定。
- **定时出报**：报表配置设置每日时间，系统自动生成昨日日报。

## 4. 功能需求（摘要）

### 4.1 报表配置（`/mappings`）

- 按当前店铺展示日报字段：规则摘要、配置/刷单设置。
- 店铺侧边栏：日期主表、定时出报、刷单统计、保存、导出 JSON。
- 与日报输出**共用**取数弹窗与刷单设置弹窗。

### 4.2 取数规则（`field_mapping_parts`）

- 文件/Sheet/列、组内多列、聚合、去重、± 组合。
- 日期列过滤、行筛选（含/不含/为空等）。
- 高级：排除样品、排除刷单、**排除当日退单**、关联数据主表有效行。
- 复用字段：`ref_field_code`（规则在基础取数字段）。

### 4.3 日报输出（`/daily`）

- 生成测试日报（`is_test`）、查看系统计算值与报表值。
- 手工行/覆盖值 PATCH 保存。
- 字段拖动排序（`sort_order`），导出按序写 Excel 列。
- 刷单模板下载与导入入口。

### 4.4 报表生成

- 输入：data_source_id、report_date、store_name。
- 输出：report_runs + report_values；支持 warning/error 状态。
- 公式：`{field:line_code}` 与四则运算（AST 安全求值）。

### 4.5 导出

- 日报 Excel：基于绑定模板，指标按 `sort_order` 从 F 列写入。
- SKU 明细：按报表日从 `fact_mc_order_sku` 导出。

## 5. 信息架构

```
AutoReport
├── 概览 (/)
├── 报表配置 (/mappings)     取数、公式、店铺设置、定时、配置导出
├── 日报输出 (/daily)          生成、编辑、排序、导出
└── （旧）模板/导入/日志      → 重定向至上述页面
```

---

## 6. 数据模型（概要）

```
catalog_* / fact_*           业务数据
data_sources.config          店铺日报上下文（主表、刷单、定时）
field_mappings + parts     报表行与取数规则
report_runs / report_values  出报结果
stores / accounts            Demo 店铺与权限
report_templates*            兼容/种子（非主路径）
```

## 10. 附录

### 10.1 启动

```bash
uvicorn app.main:app --reload --host 0.0.0.0 --port 8081
```

### 10.2 体验路径

1. 概览 → 报表配置：查看/调整取数规则  
2. 日报输出：选店与日期 → 生成 → 导出  
3. 刷单设置：导入清单 → 配置物流费 → 再生成  

### 10.3 术语

| 术语 | 定义 |
|------|------|
| 报表行 | `field_mapping`，含取数或公式 |
| line_code | 公式引用 `{field:xxx}` 的内部码 |
| 事实表 | `fact_*`，ETL 写入的店铺业务行 |
| Catalog | 文件/Sheet/列头到事实表列的映射元数据 |
